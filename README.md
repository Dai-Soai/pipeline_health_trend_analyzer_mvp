# Pipeline Health Trend Analyzer MVP

Pipeline Health Trend Analyzer is Utility #28 of the RADAR_SERVICE
Runtime Observability Phase.

It consumes historical JSON health reports generated by Utility #27
and produces structured pipeline health trend assessments.

## Status

- Version: `0.1.0`
- Release stage: MVP
- Current milestone: **M9 — Packaging & README**
- Python: `>=3.10`
- Automated tests: `92 passed`

## Purpose

Utility #27 answers:

> What is the current health of the pipeline?

Utility #28 answers:

> Is pipeline health improving, stable, or degrading over time?

The utility adds a time dimension to runtime observability by converting
multiple point-in-time health reports into a chronological trend report.

## Main capabilities

- Load one or more Utility #27 health reports
- Load historical reports from directories
- Discover reports recursively
- Validate required health report metadata
- Normalize health scores to a 0–100 scale
- Sort health reports chronologically
- Detect duplicate source run identifiers
- Convert health reports into normalized trend samples
- Calculate metric averages, minimums, maximums, deltas, and slopes
- Classify metric trends
- Determine an overall pipeline trend direction
- Generate a human-readable trend overview
- Generate JSON trend report artifacts
- Validate and inspect generated JSON reports
- Use trend thresholds in automation and CI workflows

## Trend directions

The analyzer supports four directions:

- `improving`
- `stable`
- `degrading`
- `insufficient_data`

## Default metrics

The MVP analyzes:

- `health_score`
- `warning_count`
- `critical_count`
- `total_findings`

Metric interpretation is semantic:

- Increasing health score is improving
- Decreasing warning count is improving
- Decreasing critical count is improving
- Decreasing total finding count is improving

## Core modules

### `contract.py`

Defines:

- `TrendDirection`
- `TrendSample`
- `MetricTrend`
- `TrendSummary`
- `TrendOverview`
- `TrendReport`

Contracts support:

- Validation
- Serialization
- Deserialization
- Version metadata
- Chronological sample validation
- Metric statistics validation
- Duplicate metric detection
- Summary consistency validation
- Backward-compatible optional overview enrichment

### `health_loader.py`

Loads and normalizes Utility #27 health report artifacts.

Responsibilities:

- Read JSON files
- Load multiple files
- Load directories
- Support recursive discovery
- Validate required fields
- Detect malformed JSON
- Detect duplicate `run_id` values
- Normalize health scores
- Preserve source paths and metadata
- Produce chronological `TrendSample` objects

### `trend_engine.py`

Calculates metric statistics and directions.

For each metric it produces:

- First value
- Current value
- Average value
- Minimum value
- Maximum value
- Delta
- Least-squares linear slope
- Trend direction

One sample produces:

```text
insufficient_data
```

Constant values or movements within tolerance produce:

```text
stable
```

### `analyzer.py`

Combines the loader and trend engine.

Responsibilities:

- Analyze normalized samples
- Analyze loaded reports
- Analyze explicit report paths
- Analyze a directory
- Determine overall direction
- Build `TrendSummary`
- Build complete `TrendReport`
- Generate run identifiers
- Preserve source report metadata

Overall direction policy:

- More improving metrics than degrading metrics → `improving`
- More degrading metrics than improving metrics → `degrading`
- Equal improving and degrading metrics → `stable`
- Every metric lacks sufficient history → `insufficient_data`

### `summary.py`

Builds a human-readable `TrendOverview` containing:

- Headline
- Statistical message
- Dominant direction
- Highlighted metrics
- Operational recommendation

### `report_io.py`

Provides JSON artifact support:

- Contract-aware serialization
- Contract-aware deserialization
- Atomic file writing
- Automatic parent directory creation
- JSON syntax validation
- Trend report contract validation
- Concise report inspection

### `cli.py`

Provides the installed command:

```text
pipeline-health-trend
```

## CLI usage

Display the version:

```bash
pipeline-health-trend --version
```

### Analyze historical reports

```bash
pipeline-health-trend analyze \
  examples/health_reports
```

### Display metric details

```bash
pipeline-health-trend analyze \
  examples/health_reports \
  --show-metrics
```

### Print only the direction

```bash
pipeline-health-trend analyze \
  examples/health_reports \
  --quiet
```

### Use a custom discovery pattern

```bash
pipeline-health-trend analyze \
  examples/health_reports \
  --pattern 'health_report_*.json'
```

### Discover recursively

```bash
pipeline-health-trend analyze \
  reports \
  --recursive
```

### Generate a JSON trend report

```bash
pipeline-health-trend analyze \
  examples/health_reports \
  --output reports/trend_report.json
```

### Validate a JSON trend report

```bash
pipeline-health-trend validate \
  reports/trend_report.json
```

### Inspect a JSON trend report

```bash
pipeline-health-trend inspect \
  reports/trend_report.json
```

### Use a trend threshold

```bash
pipeline-health-trend analyze \
  examples/health_reports \
  --fail-on degrading
```

Supported thresholds:

- `stable`
- `degrading`
- `insufficient_data`

CLI exit codes:

| Exit code | Meaning |
|---:|---|
| `0` | Analysis completed and threshold was not reached |
| `1` | Input, JSON, contract, or analysis error |
| `2` | Configured trend threshold was reached |

## Python API

Analyze a directory:

```python
from pipeline_health_trend_analyzer import (
    PipelineHealthTrendAnalyzer,
)

report = PipelineHealthTrendAnalyzer().analyze_directory(
    "examples/health_reports"
)

print(report.summary.overall_direction.value)
print(report.summary.sample_count)
print(report.summary.improving_count)
```

Write and inspect a JSON artifact:

```python
from pipeline_health_trend_analyzer import (
    PipelineHealthTrendAnalyzer,
    TrendReportStore,
)

report = PipelineHealthTrendAnalyzer().analyze_directory(
    "examples/health_reports"
)

store = TrendReportStore()

artifact_path = store.write(
    report,
    "reports/trend_report.json",
)

inspection = store.inspect(artifact_path)

print(inspection.overall_direction)
print(inspection.sample_count)
print(inspection.metric_count)
```
