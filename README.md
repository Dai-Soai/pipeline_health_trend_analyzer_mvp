# Pipeline Health Trend Analyzer MVP

Pipeline Health Trend Analyzer is Utility #28 of the RADAR_SERVICE
Runtime Observability Phase.

It consumes multiple historical health reports generated by Utility #27
and produces structured pipeline health trend assessments.

## Purpose

Utility #27 answers:

> What is the current health of the pipeline?

Utility #28 answers:

> Is pipeline health improving, stable, or degrading over time?

## Planned capabilities

- Load multiple JSON health reports
- Normalize chronological health samples
- Calculate score averages and deltas
- Calculate minimum and maximum values
- Calculate trend slopes
- Classify improving, stable, and degrading trends
- Summarize warning and critical finding trends
- Generate JSON trend report artifacts
- Validate and inspect generated trend reports
- Provide an automation-friendly command-line interface

## Planned trend directions

- `improving`
- `stable`
- `degrading`
- `insufficient_data`

## Trend contract

The core contract includes:

- `TrendDirection`
- `TrendSample`
- `MetricTrend`
- `TrendSummary`
- `TrendReport`

The contract supports:

- Validation
- Serialization
- Deserialization
- Chronological sample validation
- Metric statistics validation
- Trend direction aggregation
- Report and analyzer version metadata
- Summary consistency checks

## Health report loader

The health report loader consumes JSON artifacts produced by
Utility #27 — Pipeline Health Analyzer.

It supports:

- Loading one health report
- Loading multiple health reports
- Loading a directory of JSON reports
- Recursive or non-recursive discovery
- Required metadata validation
- JSON syntax validation
- Score normalization to a 0–100 scale
- Duplicate run identifier detection
- Chronological report ordering
- Conversion into validated `TrendSample` objects
- Preservation of source metadata and source paths

## Trend engine

The trend engine calculates statistical movement across chronological
health samples.

Default metrics:

- `health_score`
- `warning_count`
- `critical_count`
- `total_findings`

Calculated values:

- First value
- Current value
- Average value
- Minimum value
- Maximum value
- Delta
- Least-squares linear slope
- Trend direction

Metric direction is interpreted according to metric semantics:

- Increasing health score is improving
- Decreasing warning count is improving
- Decreasing critical count is improving
- Decreasing total finding count is improving

A single historical sample produces `insufficient_data`.
Constant values or movements within the configured tolerance produce
`stable`.
