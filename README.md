# Pipeline Health Trend Analyzer MVP

Pipeline Health Trend Analyzer is Utility #28 of the RADAR_SERVICE
Runtime Observability Phase.

It consumes multiple historical health reports generated by Utility #27
and produces structured pipeline health trend assessments.

## Purpose

Utility #27 answers:

> What is the current health of the pipeline?

Utility #28 answers:

> Is pipeline health improving, stable, or degrading over time?

## Planned capabilities

- Load multiple JSON health reports
- Normalize chronological health samples
- Calculate score averages and deltas
- Calculate minimum and maximum values
- Calculate trend slopes
- Classify improving, stable, and degrading trends
- Summarize warning and critical finding trends
- Generate JSON trend report artifacts
- Validate and inspect generated trend reports
- Provide an automation-friendly command-line interface

## Planned trend directions

- `improving`
- `stable`
- `degrading`
- `insufficient_data`

## Trend contract

The core contract includes:

- `TrendDirection`
- `TrendSample`
- `MetricTrend`
- `TrendSummary`
- `TrendReport`

The contract supports:

- Validation
- Serialization
- Deserialization
- Chronological sample validation
- Metric statistics validation
- Trend direction aggregation
- Report and analyzer version metadata
- Summary consistency checks
