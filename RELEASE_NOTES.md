# Pipeline Health Trend Analyzer MVP v0.1.0

## Overview

Pipeline Health Trend Analyzer MVP is Utility #28 of the RADAR_SERVICE
Runtime Observability Phase.

The utility consumes historical health report artifacts generated by
Utility #27 and determines whether pipeline health is improving, stable,
degrading, or lacks sufficient historical data.

## Main capabilities

- Load historical JSON health reports.
- Validate required source metadata.
- Normalize report scores to a 0–100 scale.
- Order historical samples chronologically.
- Detect duplicate source run identifiers.
- Calculate averages, minimums, maximums, deltas, and linear slopes.
- Analyze health score and finding-count trends.
- Classify each metric trend.
- Determine an overall pipeline trend.
- Generate human-readable trend summaries.
- Generate JSON trend report artifacts.
- Validate and inspect generated reports.
- Integrate trend thresholds into scripts and CI workflows.

## Installed command

```bash
pipeline-health-trend
```

## Main commands

```bash
pipeline-health-trend analyze <health-report-directory>
pipeline-health-trend validate <trend-report>
pipeline-health-trend inspect <trend-report>
```
